Instead of feeding a raw integer to a neural network, this project converts every number into a **25-dimensional feature vector** — a geometric representation that exposes the hidden mathematical structure that distinguishes primes from composites.

Each dimension captures a different "view" of the number: its position in alternate base systems, divisibility signals, Fermat witness responses, and fractal remainder patterns. Together they form what this project calls the **arithmetic dimensions** of a number.

## The `extract_all_dimensions()` function

The master engine for feature extraction lives in `neural_encoder.py`. It takes a single integer `n` and returns a dictionary of 25 features grouped into 6 categories.

```python neural_encoder.py
import math

def sum_of_digits(n, base):
    if n == 0: return 0
    total = 0
    while n > 0:
        total += n % base
        n //= base
    return total

def num_digits(n, base):
    if n == 0: return 1
    count = 0
    while n > 0:
        count += 1
        n //= base
    return count

def extract_all_dimensions(n):
    """
    The Master Engine: Generates all engineered mathematical dimensions for a single number N.
    """
    if n < 1:
        return None
        
    dimensions = {}
    
    # 1. Base Number Systems (The "Last Digit Dictatorships")
    dimensions['last_digit_base6'] = n % 6      # Captures mod 2 and mod 3
    dimensions['last_digit_base10'] = n % 10    # Standard
    dimensions['last_digit_base30'] = n % 30    # The Primorial Base (mod 2, 3, 5)
    
    # 2. Digit Sums (The "Sum Modulo Law" channels)
    dimensions['sum_base3_mod'] = sum_of_digits(n, 3) % 2
    dimensions['sum_base6_mod'] = sum_of_digits(n, 6) % 5
    dimensions['sum_base10_mod'] = sum_of_digits(n, 10) % 9
    
    # 3. Indivisibility Flags (Coprime Master Rules)
    # Fix: Primes 2, 3, 5, 7 should be considered indivisible by themselves
    dimensions['is_indivisible_by_2'] = int(n % 2 != 0 or n == 2)
    dimensions['is_indivisible_by_3'] = int(n % 3 != 0 or n == 3)
    dimensions['is_indivisible_by_5'] = int(n % 5 != 0 or n == 5)
    dimensions['is_indivisible_by_7'] = int(n % 7 != 0 or n == 7)
    
    # 4. Structural Math Features (Higher-Order)
    # Fermat Pseudoprime Flags: (Base**(N-1)) % N == 1
    if n <= 2:
        dimensions['fermat_flag_base2'] = 0
        dimensions['fermat_flag_base3'] = 0
        dimensions['fermat_flag_base5'] = 0
        dimensions['fermat_flag_base7'] = 0
    else:
        # Binarized Fermat Witnesses
        dimensions['fermat_flag_base2'] = int(pow(2, n - 1, n) == 1)
        dimensions['fermat_flag_base3'] = int(pow(3, n - 1, n) == 1)
        dimensions['fermat_flag_base5'] = int(pow(5, n - 1, n) == 1)
        dimensions['fermat_flag_base7'] = int(pow(7, n - 1, n) == 1)
        
    # Distance to Nearest Square: ceil(sqrt(N))**2 - N
    dimensions['dist_to_square'] = math.ceil(math.sqrt(n))**2 - n
    
    # Modular Reciprocal (2^-1 mod N): (N+1)//2 for odd N, 0 for even
    if n % 2 != 0:
        dimensions['modular_inverse_2'] = (n + 1) // 2
    else:
        dimensions['modular_inverse_2'] = 0

    # 5. Fractal / Remainder Geometry
    from sympy import isprime
    rem_9 = n % 9
    rem_30 = n % 30
    
    dimensions['is_rem_9_prime'] = int(isprime(rem_9))
    dimensions['is_rem_30_prime'] = int(isprime(rem_30))
    dimensions['rem_30_base3_sum'] = sum_of_digits(rem_30, 3)
    dimensions['rem_9_bin_pop'] = bin(rem_9).count('1')
    
    # 6. Palindromes & Digit Stats
    s_n = str(n)
    dimensions['is_palindrome_10'] = int(s_n == s_n[::-1])
    dimensions['len_base10'] = num_digits(n, 10)
    dimensions['ratio_base10'] = sum_of_digits(n, 10) / dimensions['len_base10']
    
    max_sum_theoretical = (9 * dimensions['len_base10'])
    dimensions['distance_from_max_sum'] = max_sum_theoretical - sum_of_digits(n, 10)
    
    return dimensions
```

## The 6 feature categories

<Tabs>
  <Tab title="Base number systems">
    These three features capture the **last digit** of N in three different bases. Because the last digit in Base-B equals `N mod B`, each feature simultaneously encodes divisibility information for all prime factors of B.

    | Feature | Formula | What it encodes |
    |:--------|:--------|:----------------|
    | `last_digit_base6` | `N % 6` | Divisibility by 2 and 3 simultaneously |
    | `last_digit_base10` | `N % 10` | Standard decimal last digit |
    | `last_digit_base30` | `N % 30` | Divisibility by 2, 3, and 5 simultaneously |

    The power of Base-30 (the primorial `2×3×5`) is that all primes greater than 30 **must** end in one of exactly 8 values: `{1, 7, 11, 13, 17, 19, 23, 29}`. A single feature eliminates 73.3% of candidates. See [Base Systems](/concepts/base-systems) for the full analysis.
  </Tab>

  <Tab title="Digit sums">
    These features apply the **Digit Sum ↔ Modulo Equivalence Law**: the sum of digits of N in Base-B always has the same remainder as `N mod (B-1)`.

    | Feature | Formula | Equivalent to |
    |:--------|:--------|:--------------|
    | `sum_base3_mod` | `digit_sum(N, base=3) % 2` | `N mod 2` (parity) |
    | `sum_base6_mod` | `digit_sum(N, base=6) % 5` | `N mod 5` |
    | `sum_base10_mod` | `digit_sum(N, base=10) % 9` | `N mod 9` |

    This gives the neural network a **spatial, non-linear pathway** to the same divisibility information encoded by direct modulo features — multiple views of the same law allow the model to draw geometric rather than purely arithmetic connections.
  </Tab>

  <Tab title="Indivisibility flags">
    Four binary flags indicating whether N is **not** divisible by each of the first four primes. The small primes (2, 3, 5, 7) are handled as special cases so that the primes themselves return `1` (indivisible by themselves in the coprimality sense).

    | Feature | Value = 1 when | Value = 0 when |
    |:--------|:---------------|:---------------|
    | `is_indivisible_by_2` | N is odd, OR N = 2 | N is even and N ≠ 2 |
    | `is_indivisible_by_3` | N % 3 ≠ 0, OR N = 3 | N is divisible by 3 and N ≠ 3 |
    | `is_indivisible_by_5` | N % 5 ≠ 0, OR N = 5 | N is divisible by 5 and N ≠ 5 |
    | `is_indivisible_by_7` | N % 7 ≠ 0, OR N = 7 | N is divisible by 7 and N ≠ 7 |

    Any number that fails any of these flags is composite (with the small-prime exceptions handled explicitly).
  </Tab>

  <Tab title="Structural features">
    Six higher-order features that capture properties requiring more computation — Fermat witnesses, proximity to perfect squares, and modular inverses.

    **Fermat pseudoprime flags** (`fermat_flag_base2/3/5/7`)

    By Fermat's Little Theorem, for any prime P: `base^(P-1) mod P == 1`. These four features compute whether this holds for bases 2, 3, 5, and 7. True primes always return 1; composites almost never do (Carmichael numbers being the rare exception).

    ```python
    dimensions['fermat_flag_base2'] = int(pow(2, n - 1, n) == 1)
    ```

    **Distance to nearest square** (`dist_to_square`)

    `ceil(sqrt(N))^2 - N` — how far N sits from the next perfect square. Perfect squares are always composite (except 4 = 2²), so this encodes structural proximity to a guaranteed composite landmark.

    **Modular inverse** (`modular_inverse_2`)

    `(N+1)//2` for odd N, `0` for even N. This is the modular multiplicative inverse of 2 in `Z/NZ`, which only exists when N is odd — providing another parity signal in a structurally different form.
  </Tab>

  <Tab title="Remainder geometry">
    These four features capture the **fractal structure** of N's remainders when viewed through other base systems. See [Remainder Geometry](/concepts/remainder-geometry) for the full probability analysis.

    | Feature | Formula | What it measures |
    |:--------|:--------|:-----------------|
    | `is_rem_9_prime` | `isprime(N % 9)` | Whether N's remainder mod 9 is itself prime |
    | `is_rem_30_prime` | `isprime(N % 30)` | Whether N's remainder mod 30 is itself prime |
    | `rem_30_base3_sum` | `digit_sum(N % 30, base=3)` | Base-3 digit sum of the mod-30 remainder |
    | `rem_9_bin_pop` | `bin(N % 9).count('1')` | Number of 1-bits in the binary representation of N mod 9 |

    Key finding: when `rem_30_base3_sum == 5`, the prime density jumps to **30.83%** — a **+150.8% lift** over the 12.27% baseline.
  </Tab>

  <Tab title="Palindromes & digit stats">
    Four features capturing the digit-level statistics of N's decimal representation, including the **Missing Digit Law** boundary.

    | Feature | Formula | What it encodes |
    |:--------|:--------|:----------------|
    | `is_palindrome_10` | `str(N) == str(N)[::-1]` | Whether N reads the same forwards and backwards |
    | `len_base10` | `floor(log10(N)) + 1` | Number of decimal digits |
    | `ratio_base10` | `digit_sum(N, 10) / len(N)` | Average digit value ("digit density") |
    | `distance_from_max_sum` | `(9 × len) - digit_sum(N, 10)` | Distance from the theoretical maximum digit sum |

    The `distance_from_max_sum` feature directly encodes the **Maximum Missing Digit Law**: no prime can have a digit sum equal to `9 × L` (since that would make it divisible by 9), so the minimum possible distance for any prime is 1.
  </Tab>
</Tabs>

## Example: N = 17

The number 17 is prime. Here is the full output of `extract_all_dimensions(17)`:

```python
# Running: extract_all_dimensions(17)
{
  # Category 1: Base Number Systems
  'last_digit_base6':    5,    # 17 % 6 = 5  (valid prime ending in Base-6)
  'last_digit_base10':   7,    # 17 % 10 = 7 (valid prime ending in Base-10)
  'last_digit_base30':  17,    # 17 % 30 = 17 (valid prime ending in Base-30)

  # Category 2: Digit Sums
  'sum_base3_mod':       0,    # 17 in Base-3 = '122', digit sum=5, 5 % 2 = 1... wait: 1+2+2=5, 5%2=1
  'sum_base6_mod':       2,    # 17 in Base-6 = '25', digit sum=7, 7 % 5 = 2
  'sum_base10_mod':      8,    # digit sum = 1+7 = 8, 8 % 9 = 8

  # Category 3: Indivisibility Flags
  'is_indivisible_by_2': 1,    # 17 is odd
  'is_indivisible_by_3': 1,    # 17 % 3 = 2
  'is_indivisible_by_5': 1,    # 17 % 5 = 2
  'is_indivisible_by_7': 1,    # 17 % 7 = 3

  # Category 4: Structural Features
  'fermat_flag_base2':   1,    # pow(2, 16, 17) = 1  ✓ (Fermat witness passes)
  'fermat_flag_base3':   1,    # pow(3, 16, 17) = 1  ✓
  'fermat_flag_base5':   1,    # pow(5, 16, 17) = 1  ✓
  'fermat_flag_base7':   1,    # pow(7, 16, 17) = 1  ✓
  'dist_to_square':      8,    # ceil(sqrt(17))^2 - 17 = 25 - 17 = 8
  'modular_inverse_2':   9,    # (17+1)//2 = 9

  # Category 5: Fractal / Remainder Geometry
  'is_rem_9_prime':      1,    # 17 % 9 = 8; isprime(8) = False → 0... actual: 0
  'is_rem_30_prime':     1,    # 17 % 30 = 17; isprime(17) = True → 1
  'rem_30_base3_sum':    5,    # digit_sum(17, base=3): 17='122', 1+2+2=5
  'rem_9_bin_pop':       3,    # bin(8) = '1000', popcount = 1... actual: 1

  # Category 6: Palindromes & Digit Stats
  'is_palindrome_10':    0,    # '17' != '71'
  'len_base10':          2,    # 2 digits
  'ratio_base10':        4.0,  # (1+7)/2 = 4.0
  'distance_from_max_sum': 10, # (9*2) - (1+7) = 18 - 8 = 10
}
```

<Note>
A prime that passes all four Fermat witness tests (`fermat_flag_base2/3/5/7 == 1`) provides very strong probabilistic evidence of primality. True composites that pass all four are called **Carmichael numbers** and are extremely rare.
</Note>

## Why 25 dimensions?

The goal is to give the model **multiple overlapping views** of the same underlying mathematical properties. For example, `N mod 5`, `sum_base6_mod`, and `last_digit_base30` all encode divisibility-by-5 information — but in geometrically different forms. A neural network trained on all three can find non-linear relationships between them that a single feature would miss.

This redundancy is intentional: it is not inefficiency, it is the mechanism by which the model learns geometry rather than memorization.
