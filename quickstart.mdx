Get the Prime Number Neural Network running locally and make your first prime prediction.

<Steps>

<Step title="Install dependencies">

The project requires Python 3.8+ and four libraries. Install them with pip:

```bash
pip install pandas numpy scikit-learn sympy
```

<Note>
  `sympy` is used for ground-truth prime verification via `isprime()`, and also inside `extract_all_dimensions()` to compute fractal remainder features.
</Note>

</Step>

<Step title="Explore a single number's dimensions">

Before training a model, you can inspect the raw feature vector that the encoder produces for any integer. Import `extract_all_dimensions` from `neural_encoder.py` and call it directly:

```python neural_encoder_demo.py
from neural_encoder import extract_all_dimensions

dims = extract_all_dimensions(97)
for key, value in dims.items():
    print(f"{key}: {value}")
```

Expected output for `n = 97`:

```

last_digit_base10: 7
last_digit_base30: 7
sum_base3_mod: 1
sum_base6_mod: 1
sum_base10_mod: 7
is_indivisible_by_2: 1
is_indivisible_by_3: 1
is_indivisible_by_5: 1
is_indivisible_by_7: 1
fermat_flag_base2: 1
fermat_flag_base3: 1
fermat_flag_base5: 1
fermat_flag_base7: 1
dist_to_square: 3
modular_inverse_2: 49
is_rem_9_prime: 0
is_rem_30_prime: 1
rem_30_base3_sum: 3
rem_9_bin_pop: 3

len_base10: 2

```

<Tip>
  All four `fermat_flag_base*` values being `1` is a strong signal that 97 is prime — Fermat's little theorem holds for true primes across every base.
</Tip>

The function returns `None` for `n < 1`, so always validate your input before encoding:

```python
dims = extract_all_dimensions(0)   # returns None
dims = extract_all_dimensions(-5)  # returns None
```

</Step>

<Step title="Train the two-stage ensemble">

`neural_net.py` builds and evaluates the full ensemble. It generates a dataset from `1` to `20,000`, trains on the first half, and extrapolates to the second half without any random shuffling:

<CodeGroup>

```bash run directly
python neural_net.py
```

```python import and call
from neural_net import build_ensemble

build_ensemble()
```

</CodeGroup>

`build_ensemble()` runs two training phases in sequence:

**Phase 1 — Base-System Filter (Model A)**

A shallow `DecisionTreeClassifier` (depth 5) that eliminates obvious composites using modular arithmetic features:

```python neural_net.py
filter_features = [
    'last_digit_base6', 'last_digit_base30',
    'sum_base3', 'sum_base6', 'sum_base10',
    'is_even_length_palindrome'
]

model_a = DecisionTreeClassifier(max_depth=5, random_state=42)
model_a.fit(X_train_filter, y_train)
```

**Phase 2 — Deep Neural Network (Model B)**

An `MLPClassifier` with three hidden layers (`64 → 128 → 64`) that resolves the ambiguous candidates that Model A passed through:

```python neural_net.py
complex_features = [
    'len_base2', 'sum_base2', 'ratio_base2', 'ratio_base10',
    'distance_from_max_sum', 'last_digit_base210', 'last_digit_base30'
]

model_b = MLPClassifier(
    hidden_layer_sizes=(64, 128, 64),
    activation='relu',
    max_iter=1000,
    random_state=42
)
model_b.fit(X_train_dnn, y_train_dnn)
```

<Warning>
  Dataset generation calls `extract_all_dimensions()` and `isprime()` for every number from 1 to 20,000. On a typical laptop this takes 1–3 minutes. The `sympy.isprime` calls dominate runtime for numbers with many Fermat checks.
</Warning>

</Step>

<Step title="Run the full evaluation suite">

`final_ensemble_test.py` extends the evaluation to three extrapolation ranges: `10k–20k`, `50k–60k`, and `100k–110k`. This tests whether the learned features generalize far beyond the training distribution.

<CodeGroup>

```bash run directly
python final_ensemble_test.py
```

```python import and call
from final_ensemble_test import run_evaluation

run_evaluation()
```

</CodeGroup>

`run_evaluation()` trains on `1–10,000` using the refined feature sets and threshold tuning from `final_ensemble_test.py`:

```python final_ensemble_test.py
filter_features = [
    'last_digit_base6', 'last_digit_base30',
    'is_indivisible_by_2', 'is_indivisible_by_3',
    'is_indivisible_by_5', 'is_indivisible_by_7'
]

complex_features = [
    'fermat_flag_base2', 'fermat_flag_base3',
    'fermat_flag_base5', 'fermat_flag_base7',
    'dist_to_square', 'modular_inverse_2',
    'is_rem_9_prime', 'is_rem_30_prime',
    'rem_30_base3_sum', 'rem_9_bin_pop',
    'ratio_base10', 'distance_from_max_sum', 'is_palindrome_10'
]
```

Model A uses a class weight of `{0: 20, 1: 1}` to heavily penalize false positives on composites, and Model B uses a high-confidence threshold of `0.9` for its prime predictions:

```python final_ensemble_test.py
model_a = DecisionTreeClassifier(
    max_depth=5,
    class_weight={0: 20, 1: 1},
    random_state=42
)

# Model B prediction with precision threshold
b_preds = (b_probs > 0.9).astype(int)
```
