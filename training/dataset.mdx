The ensemble is trained on integers from 1 to 20,000. Every integer in that range is encoded into a feature vector by `extract_all_dimensions()`, then labeled as prime or composite using `sympy.isprime`.

## Data generation

The `generate_range()` function in `final_ensemble_test.py` is the canonical data builder. It calls `extract_all_dimensions(i)` for every integer in a range and appends the prime label:

```python neural_net.py
data = []
for i in range(1, 20001):
    dims = extract_all_dimensions(i)
    dims['is_prime'] = int(isprime(i))
    dims['number'] = i
    data.append(dims)

df = pd.DataFrame(data)
```

The reusable version in `final_ensemble_test.py`:

```python final_ensemble_test.py
def generate_range(start, end):
    print(f"Generating data for {start} to {end}...")
    data = []
    for i in range(start, end + 1):
        dims = extract_all_dimensions(i)
        if dims is None: continue
        dims['is_prime'] = int(isprime(i))
        dims['number'] = i
        data.append(dims)
    return pd.DataFrame(data)
```

`sympy.isprime` is called once per integer. This is the dominant runtime cost — see [Running Training](/training/running-training) for timing guidance.

## Why random splits are banned

<Note>
  `sklearn.model_selection.train_test_split` is not used anywhere in this project. Using it would invalidate all accuracy metrics.
</Note>

Prime numbers cluster in ways that make random shuffling dangerous. If you split randomly, a number like 10,007 (prime) could end up in the training set while 10,009 (prime) ends up in the test set. The model learns to recognize the neighborhood of 10,007 and achieves high apparent accuracy by interpolation rather than structural understanding.

The goal of this project is to demonstrate that the model has learned *structural* mathematical laws — patterns encoded in base representations, Fermat pseudoprime flags, and remainder geometry — that generalize to arbitrarily large integers the model has never seen.

## Temporal split

The split is strict and ordinal:

| Partition | Range | Purpose |
|-----------|-------|---------|
| Train | 1 – 10,000 | Model fitting |
| Test | 10,001 – 20,000 | Extrapolation evaluation |

```python neural_net.py
train_df = df[df['number'] <= 10000].copy()
test_df  = df[df['number'] > 10000].copy()
```

The model never sees any number above 10,000 during training. A correct prediction on 100,000–110,000 can only come from generalized structural knowledge, not memorization.

## Extrapolation evaluation ranges

`run_evaluation()` in `final_ensemble_test.py` tests the trained ensemble against four ranges:

```python final_ensemble_test.py
train_df  = generate_range(1,      10000)   # training
test_20k  = generate_range(10001,  20000)   # near extrapolation
test_60k  = generate_range(50000,  60000)   # far extrapolation
test_110k = generate_range(100000, 110000)  # infinite-scale test
```

Successfully maintaining accuracy at 100k–110k is the key benchmark: it means the model's learned rules still hold an order of magnitude beyond the training distribution.

## Feature generation

Every row in the dataset is produced by `extract_all_dimensions(n)` in `neural_encoder.py`. The function returns a flat dictionary of engineered dimensions grouped into six categories:

| Category | Example features |
|----------|------------------|
| Base last digits | `last_digit_base6`, `last_digit_base30` |
| Digit sums | `sum_base3_mod`, `sum_base6_mod`, `sum_base10_mod` |
| Indivisibility flags | `is_indivisible_by_2/3/5/7` |
| Structural / higher-order | `fermat_flag_base2/3/5/7`, `dist_to_square`, `modular_inverse_2` |
| Fractal remainder geometry | `is_rem_9_prime`, `is_rem_30_prime`, `rem_30_base3_sum`, `rem_9_bin_pop` |
| Digit statistics | `is_palindrome_10`, `ratio_base10`, `distance_from_max_sum` |

No raw divisibility checks (e.g., `n % 11 == 0`) are used as features. All features are mathematical properties computable for any integer.

## Class imbalance

Primes become increasingly sparse as integers grow larger. In the 1–10,000 training range roughly 12.3% of integers are prime (there are 1,229 primes below 10,000). This is the training-time class distribution the models see.

Without correction, a classifier could reach ~87.7% accuracy by predicting every number as composite. To counteract this, the final ensemble applies heavy class weighting in Model A:

```python final_ensemble_test.py
# Penalize False Positives 20× more than False Negatives.
# This forces the tree to be certain before calling a number composite.
model_a = DecisionTreeClassifier(
    max_depth=5,
    class_weight={0: 20, 1: 1},
    random_state=42
)
```

The weight `{0: 20, 1: 1}` means the cost of misclassifying a composite as prime (false positive) is twenty times the cost of misclassifying a prime as composite (false negative). This makes Model A conservative: it passes ambiguous candidates to Model B rather than dismissing them.
